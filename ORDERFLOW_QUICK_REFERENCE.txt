QUICK START (3 STEPS):

REST API ENDPOINTS:
================================================================================

All endpoints start with: /api/orderflow/

GET /snapshot?symbol=SPY
  → Last 100 trades with current bid/ask

GET /stats?symbol=SPY&time_window=60
  → Volume, delta, VWAP, trade count (60 second window)

GET /large-trades?symbol=SPY&min_size=10000
  → Recent large trades and institutional sweeps

GET /volume-profile?symbol=SPY&time_window=300
  → Volume by price level, POC, value area

GET /delta-chart?symbol=SPY&interval_seconds=1
  → Delta over time in 1-second buckets

GET /heat-map?symbol=SPY&price_precision=0.05
  → Time-price intensity grid for heatmap visualization

POST /start?symbol=SPY
  → Start streaming for symbol

POST /stop?symbol=SPY
  → Stop streaming

POST /clear?symbol=SPY
  → Clear cached data

GET /status
  → Check which symbols are actively streaming

GET /health
  → Health check endpoint

CONNECTING REAL DATA:
================================================================================

Pattern 1: Polygon.io WebSocket
──────────────────────────────
  from data.providers.orderflow_stream import process_polygon_trade
  import websockets

  async def polygon_client(api_key):
      async with websockets.connect(f"wss://socket.polygon.io/stocks?apiKey={api_key}") as ws:
          await ws.send(json.dumps({"action": "subscribe", "params": "T.SPY,T.SPX"}))
          async for message in ws:
              for trade in json.loads(message).get("data", []):
                  await process_polygon_trade(trade)

  # In app startup:
  @app.on_event("startup")
  async def startup():
      asyncio.create_task(polygon_client("YOUR_API_KEY"))


Pattern 2: Custom Data Source
─────────────────────────────
  from data.providers.orderflow_stream import orderflow_stream

  async def your_data_source():
      async for trade in receive_trades():
          await orderflow_stream.process_trade({
              "symbol": trade.symbol,
              "price": trade.price,
              "size": trade.size,
              "timestamp": trade.timestamp.isoformat() + "Z",
              "bid": trade.bid,
              "ask": trade.ask,
              "conditions": trade.conditions
          })


TRADE CLASSIFICATION (TICK RULE):
================================================================================

How the system determines if a trade is aggressive buy/sell:

  Rule 1: Price ≈ Ask Price → BUY
           (Buyer is aggressive, lifting the ask)

  Rule 2: Price ≈ Bid Price → SELL
           (Seller is aggressive, hitting the bid)

  Rule 3: Price between bid/ask → Use last price comparison
           (Compare to previous trade price)

  Rule 4: Unable to classify → NEUTRAL

Example:
  Bid: 559.21, Ask: 559.23, Current Trade: 559.23 → BUY
  Bid: 559.21, Ask: 559.23, Current Trade: 559.21 → SELL
  Bid: 559.21, Ask: 559.23, Current Trade: 559.22 → NEUTRAL


COMMON CUSTOMIZATIONS:
================================================================================

Change bubble appearance:
  CONFIG.BUBBLE_MAX_RADIUS = 40        // Larger bubbles
  CONFIG.BUBBLE_FADE_TIME = 12000      // Longer visibility

Change colors:
  In orderflow.html, find getColorForSide() and modify:
    'buy': '#00ff88'   (green)
    'sell': '#ff4444'  (red)
    'neutral': '#888'  (gray)

Change animation speed:
  In Bubble class constructor, modify:
    200 ms = entrance animation duration
    8000 ms = total lifetime before fade

Change grid spacing (price levels):
  CONFIG.PRICE_GRID_SPACING = 0.05   (SPY: $0.05)
  CONFIG.PRICE_GRID_SPACING = 0.50   (SPX: $0.50)

Change time window:
  CONFIG.TIME_WINDOW_SECONDS = 120    (show 2 minutes of data)

TROUBLESHOOTING:
================================================================================

Problem: "No data showing"
  Solution: Click "Demo" button to enable simulated data

Problem: "WebSocket connection failed"
  Solution: Check if FastAPI server is running on port 8000
  Solution: Check browser console for error messages

Problem: "Performance is slow"
  Solution: Reduce CONFIG.MAX_BUBBLES (5000 → 2000)
  Solution: Reduce CONFIG.TIME_WINDOW_SECONDS (60 → 30)

Problem: "Bubbles disappear too fast"
  Solution: Increase CONFIG.BUBBLE_FADE_TIME (8000 → 15000)

Problem: "Charts look frozen"
  Solution: Check WebSocket connection status in header
  Solution: Refresh the page
