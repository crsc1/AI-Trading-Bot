API ENDPOINTS (14 TOTAL)
========================

SIGNALS:
  GET  /api/signals           → Get signals (filter by symbol/direction/confidence)
  GET  /api/signals/latest    → Get most recent signal

TRADES:
  GET  /api/trades            → Get trade history (date range)

P&L:
  GET  /api/pnl               → Daily P&L summary
  GET  /api/pnl/history       → 30-day P&L history (for charting)

MARKET DATA:
  GET  /api/market            → SPY, SPX, VIX, QQQ prices

OPTIONS FLOW:
  GET  /api/flow              → Unusual options activity (sweeps, blocks)

BOT STATUS:
  GET  /api/status            → Bot status (running/mode/positions/P&L)

TRADE CONTROL:
  POST /api/trade/approve/{id}→ Execute a signal (semi-auto)
  POST /api/trade/reject/{id} → Reject a signal

SETTINGS:
  POST /api/settings          → Update bot configuration

ANALYSIS:
  GET  /api/analysis/yesterday→ Why did SPY move yesterday?
  GET  /api/patterns          → Current pattern analysis
  GET  /api/calendar          → Economic events calendar

WEBSOCKET MESSAGE TYPES
=======================

connected               Initial connection established
new_signal              New trading signal generated
trade_executed          Trade was executed
price_update            Market price changed
pnl_update              Profit/loss updated
options_flow            New unusual options activity
status_update           Bot status changed
heartbeat               Keep-alive (every 30 seconds)

AUTO-REFRESH INTERVALS
======================

Market data:   5 seconds  (setInterval(fetchMarketData, 5000))
All data:      30 seconds (setInterval(loadAllData, 30000))

WebSocket:     30 seconds (heartbeat keep-alive)
Reconnect:     Exponential backoff (1s → 30s max)

INTEGRATION STEPS
=================

1. REPLACE MOCK DATA
   In api_routes.py, each endpoint has # TODO: comments
   Replace mock return values with real engine calls

2. IMPLEMENT STARTUP HOOKS
   In app.py @app.on_event("startup"):
   - Connect to trading engine
   - Initialize data feeds
   - Start background monitoring

3. BROADCAST REAL-TIME UPDATES
   From your trading engine:
   await manager.broadcast_signal({...})
   await manager.broadcast_trade_execution({...})
   await manager.broadcast_price_update({...})

4. TEST INTEGRATION
   Run dashboard and verify:
   - Signals appear and update
   - Trades are logged
   - P&L updates in real-time
   - Charts display correctly

TROUBLESHOOTING
===============

Problem: Dashboard shows "Loading..."
Solution: Check browser console (F12) for errors
         Verify API endpoints return data
         Check server: http://localhost:8000/health

Problem: WebSocket not connecting
Solution: Verify server running (http://localhost:8000)
         Check firewall port 8000
         Look for connection errors in console

Problem: Charts not displaying
Solution: Check Chart.js CDN loaded in Network tab
         Verify pnl_chart canvas exists
         Check P&L data returned from API

Problem: Real-time updates not working
Solution: Check WebSocket status badge (green = connected)
         Verify trading engine broadcasting updates
         Check browser console for JavaScript errors

CUSTOMIZATION
==============

CHANGE COLORS:

PRODUCTION DEPLOYMENT
=====================

Single worker:
$ uvicorn app:app --host 0.0.0.0 --port 8000

Multiple workers (4x):
$ uvicorn app:app --host 0.0.0.0 --port 8000 --workers 4

With systemd (Linux):
  Create /etc/systemd/system/trading-dashboard.service
  Edit ExecStart line with full path to Python
  systemctl start trading-dashboard
  systemctl enable trading-dashboard

With Docker:
  Create Dockerfile with Python base image
  COPY dashboard/ /app/
  RUN pip install -r requirements.txt
  CMD ["uvicorn", "app:app", "--host", "0.0.0.0"]

With Nginx reverse proxy:
  upstream dashboard {
    server localhost:8000;
  }
  server {
    listen 80;
    location / {
      proxy_pass http://dashboard;
      proxy_http_version 1.1;
      proxy_set_header Upgrade $http_upgrade;
      proxy_set_header Connection "upgrade";
    }
  }
